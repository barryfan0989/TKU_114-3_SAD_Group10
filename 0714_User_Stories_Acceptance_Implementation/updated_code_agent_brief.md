# 更新後的程式碼代理實作任務書 (Updated Code Agent Brief)

本文件是給程式碼代理（Code Agent）的實作規格書，詳細定義了本次第二個可操作切片的開發範圍、保留項目、新增/修正項目與驗收標準。

---

## 一、專案名稱
吉他社社費預約繳費與電子收據管理系統 - 第二個可操作切片（時效模擬、櫃檯現場變更、社長關懷名冊與收據大印列印優化）

## 二、目前系統狀態
已完成第一個可操作切片：
1. 社員可線上預約（半年500/一年900），產生唯一 RES- 編號並顯示待繳卡片。
2. 財務後台具備模糊搜尋（< 0.5s）與「一鍵確認收款」按鈕。點擊後狀態變為已繳費，並自動記錄經辦人「財務幹部 林同學」與付款時間。
3. 社長看板能統計繳費率、實收金額、待收金額，並分類列出已繳/未繳名單。
4. 社員能以學號查詢電子收據憑證。
5. 資料以 LocalStorage 儲存，預載 20 筆測試資料。

## 三、本次目標
實作「第二個可操作切片」，優化繳費現場靈活性、使自動逾期（BR-01）可在現場被模擬測試，並補強社長催繳關懷手段及收據合規評鑑印章與列印樣式。

## 四、保留項目 (不可破壞)
1. 頂部身分切換機制與 SPA 視圖切換。
2. 既有 LocalStorage 的資料讀寫引擎與預載的 20 筆社員測試資料結構。
3. 手寫的簡約 Notion 紙質視覺風格（hairline 細邊框、克制的使用藍色與灰色）。
4. 繁體中文介面文字。

## 五、修正項目
* **過期資料統計邏輯修正 (BR-01)**：已逾期的預約單（狀態為「已逾期」），其金額必須完全從社長看板的「待收金額」中扣除，且不計入已繳費率的分母。

## 六、新增項目 (第二切片核心)
1. **系統日期模擬器 (Date Simulator - DR-03)**：
   - 於頂部導覽列（身分切換按鈕旁）新增一個模擬系統日期輸入框（`<input type="date">`），預設值為今日實際日期，並同步存入 LocalStorage。
   - 當頁面載入，或是使用者變更模擬日期時，系統應掃描所有資料。如果某筆資料狀態為 `待繳費` 且其 `dateCreated` 與模擬日期的差距大於 7 天（即差距 >= 8 天），則自動將其狀態變更為 `已逾期` 並存回 LocalStorage，隨即彈出 Toast 提示。
2. **櫃檯現場方案快速變更 (Quick Scheme Editor - FR-08)**：
   - 於財務幹部後台資料表中，針對狀態為 `待繳費` 的社員資料列，在「繳費方案」欄位中提供一個方案下拉切換選單（半年期 500元 / 一年期 900元）。
   - 財務幹部可現場為學員直接點選變更方案，變更後表格中的金額即時更新，LocalStorage 資料同步被修改。
   - 後續點擊「一鍵確認收款」將直接以此方案及金額結帳並開立收據。
3. **社長逾期關懷名冊 (President Expiration Care List - BR-01)**：
   - 社長後台的未繳費清單拆分為兩個獨立的清單卡片：
     - **待繳費社員清冊** (狀態為待繳費且未逾期)。
     - **逾期關懷名冊 (逾期未繳社員清單)** (狀態為已逾期)。
   - 社長可直接查閱逾期名單，以便線下進行催繳或關懷。
4. **電子收據大印與列印樣式優化 (Receipt Stamp & Print - FR-07)**：
   - 於渲染電子收據時，加蓋紅色的圓形社團財務認證收訖大印（以 CSS 浮水印或 SVG 在收據背景呈現，字樣為「淡江大學吉他社 財務收訖章」）。
   - 在 CSS 中實作列印樣式（`@media print`），點擊收據底部的「列印/下載收據憑證」按鈕觸發 `window.print()` 時，自動隱藏系統的頂部導覽按鈕、身份切換、搜尋框與列印按鈕，只保留收據票據本體滿版呈現。

## 七、刪除項目 (防膨脹)
* 移除任何可能出現在程式碼中的真實登入表單、外部金流串接預留代碼、Email/簡訊發送 API 連線。

## 八、本次不做 (範圍控制)
1. 真實後端伺服器與 SQL 資料庫連線。
2. 真實帳號註冊與密碼認證。
3. 行動支付（Line Pay）或信用卡線上交易 API。
4. 自動發送簡訊/Email 給社員。

## 九、技術與資料限制
* 限用原生 HTML5、Vanilla CSS、Vanilla JS。不引入 React、Vue 等前端框架，亦不引入 Bootstrap、TailwindCSS 等外部 CSS 樣式庫。
* 所有資料必須維持在 LocalStorage 中持久化。

## 十、對應需求與驗收條件
* **FR-03, FR-04, FR-05, FR-07, FR-08**
* **BR-01, BR-02, DR-01, DR-02, DR-03**
* **NFR-01, NFR-02, NFR-03, NFR-05**
* **AC-08-02, AC-09-01, AC-09-02, AC-10-01**

## 十一、實作順序
1. 先修改 `index.html`，在頂部身分切換旁新增模擬日期輸入欄。在財務後台表格列中，為「待繳費」狀態預留下拉選單。在社長看板下拆分出逾期名冊。在收據結構中增設印章浮水印。
2. 修改 `app.js`：
   - 新增 LocalStorage 日期初始化與時間掃描逾期邏輯。
   - 新增方案快速切換的事件監聽與資料更新。
   - 修正社長儀表板統計，將已逾期的金額排除在待收金額之外，並分類渲染出社長雙名冊。
   - 渲染電子收據時，同步蓋上社團紅印。
3. 修改 `style.css`：加入模擬時間器、方案切換選單、社長雙名冊及電子收據印章的 Notion 質感樣式，並設定 `@media print` 列印媒體查詢規則。
4. 進行手動測試與驗收。

## 十二、不可自行決定的事項
* 不可自行更改半年 500 元 / 一年 900 元的限制金額。
* 不可自行增加「課外活動組」等不存在的角色頁面。
* 不可自行改用 Express / Node.js 或引入外部伺服器。
