# 程式碼代理提示詞紀錄 (Code Agent Prompt Log)

本文件紀錄了開發「社費預約繳費與電子收據管理系統第一個可操作切片」時，與程式碼代理交互的提示詞（Prompts）、AI 產出重點及人工微調修正紀錄。

| 次數 | 使用目的 | 提示詞摘要 (Prompt Summary) | 產出摘要 (Output Summary) | 採用內容 | 不採用內容 | 人工修正 (Manual Adjustments) |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **1** | 建立實作計畫與檔案結構 | 提供專案背景與技術限制，要求代理規劃實作計畫，建立第一個可操作切片。 | 提出單頁應用（SPA）架構計畫，包含 `index.html`（HTML5 結構）、`style.css`（現代玻璃擬態設計樣式表）與 `app.js`（LocalStorage 狀態與模擬資料管理）。 | 採用全盤計畫與檔案目錄架構。 | 無。 | 無。 |
| **2** | 實作 UI 介面與預載資料 | 依據實作計畫，要求代理建立 `index.html` 與現代化 `style.css`。要求具備頂部身分切換與響應式佈局。 | 產出帶有精美漸層背景、卡片懸停效果與 RWD 切換的網頁。預留了社員表單、財務搜尋表格、社長統計看板之 HTML 框架。 | 採用 HTML 架構與 CSS 視覺設計（包含深色模式配色、Inter 字型與玻璃擬態按鈕樣式）。 | 無。 | 修正 CSS 容器間距，使其在窄螢幕（手機）下的對齊更加美觀，避免按鈕擠壓變形。 |
| **3** | 實作互動邏輯與 LocalStorage | 要求代理實作 `app.js`，包含 20 筆社員預載測試資料初始化、社員送出預約（自動生成 RES 編號）、財務即時模糊搜尋、一鍵確認收款（紀錄時間戳記與經辦人林同學）、社長統計即時更新以及收據學號查詢。 | 產出完整的 LocalStorage 狀態管理 JavaScript 程式碼。實現了即時模糊過濾演算法、社長看板數據動態加值計算，以及收據模組。 | 採用全部 LocalStorage 儲存、時間戳記紀錄及模糊搜尋匹配邏輯。 | 原先代理在社員送出預約後，會將狀態卡片自動渲染至財務版面。由於是 SPA 切換，為保持角色獨立性，修改為僅更新 LocalStorage，各角色在切換時重新讀取。 | 1. 修正 `BR-01` 逾期資料統計：確保狀態為「已逾期」的預約項目之金額不會被計入社長的「已收總額」中。<br>2. 微調日期生成格式，使其與現場測試時間對齊。 |
