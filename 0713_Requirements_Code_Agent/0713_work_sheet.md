## C. 訪談大綱 (Interview Guide)

本大綱針對一般社員、財務幹部、社長設計了對應的背景、流程、痛點、例外及成功指標之訪談問題，已整理存檔於 [interview_guide.md](file:///c:/Users/USER/Desktop/3333/TKU_114-3_SAD_Group10/0713_Requirements_Code_Agent/interview_guide.md)。

### 角色 1：一般社員 (General Member)
* **受訪角色**：一般社員 (蔡同學)
* **角色與系統關係**：預約繳費人與收據查驗人。
* **訪談目的**：確認社員繳納社費流程的滿意度、痛點與電子收據的需求。
* **主要問題**：包含背景、流程痛點、預約期限（ BR-01 ）、預約成功指示與首頁卡片體驗等 6 題。

### 角色 2：財務幹部 (Financial Officer)
* **受訪角色**：財務幹部 (林同學)
* **角色與系統關係**：現場現金收款確認操作者。
* **訪談目的**：確認現場收款的瓶頸、模糊搜尋精準度、一鍵確認的操作便利性及責任防弊機制。
* **主要問題**：包含收款人數、Google 試算表記帳痛點、方案現場變更需求、防誤點機制與搜尋效能等 6 題。

### 角色 3：社長 (Club President)
* **受訪角色**：社長 (范同學)
* **角色與系統關係**：社團財務狀況檢視與決策管理者。
* **訪談目的**：確認社長對於缴費率、實收總額、未繳費清冊及評鑑憑證的法規與視覺化需求。
* **主要問題**：包含財務匯報頻率、催繳痛點、收支對帳稽核、收據欄位法規要求等 6 題。

### 1. 專案背景
針對吉他社社費繳交流程進行優化，提供預約、搜尋、一鍵確認、收據憑證、社長看板之無紙化前端單頁原型。

### 2. 目標角色
* **一般社員**：預約方案、取得編號、查驗收據。
* **財務幹部**：即時模糊搜尋、一鍵收款登記。
* **社長**：即時繳費率、已收與待收金額、已繳/未繳名冊。

### 3. 第一個可操作切片
「社員選擇方案送出預約，財務幹部在後台模糊搜尋並一鍵確認收款，系統隨即生成電子收據；社長後台即時更新繳費率統計與已繳/未繳名冊。」

### 4. 本期範圍內項目
身分切換、預約方案 (500/900)、RES編號生成、首頁狀態卡片、後台模糊搜尋、一鍵收款登記、帶經辦人的電子收據查驗、社長即時儀表板與 20 筆社員預載資料。

### 5. 本期範圍外項目
真實後端伺服器與資料庫（改用 LocalStorage）、真實登入註冊密碼、真實線上金流 API、自動 Email/SMS 外部發送服務、退費審核。

### 6. 本次要實作的需求
* **FR-01 至 FR-07**
* **BR-01, NFR-01, NFR-02, CON-01**
（驗收條件詳見 `code_agent_implementation_brief.md` 第六節）

### 7. 技術限制
* **前端技術**：原生 HTML5, Vanilla CSS (手寫質感 Notion 紙質簡約樣式)
* **後端技術**：無。
* **資料儲存**：LocalStorage。
* **部署與登入**：無需正式部署，不含真實登入，使用身分切換按鈕。
* **外部套件**：僅允許引入 FontAwesome 圖標庫與 Google Fonts。

### 8. 程式碼代理不可自行決定的事項
不可新增未定義角色（如維修商）；不可修改社費方案金額；不可加裝真實登入註冊；不可引入外部 UI 框架（如 Bootstrap / Tailwind）。

### 9. 若代理想加入，必須拒絕的功能
* **自動 Email 發送**：拒絕，理由是增加外部服務串接複雜度與時程風險。
* **Line Pay 金流**：拒絕，涉及真實金融安全與學校行政法規。

### 10. 交給程式碼代理的正式任務文字
已在任務書第十節中提供，用於啟動實作原型。

## H. 程式碼代理提示詞紀錄 (Code Agent Prompt Log)

本組在實作過程中的提示詞與 AI 交互紀錄已詳細整理並存檔於 [code_agent_log.md](file:///c:/Users/USER/Desktop/3333/TKU_114-3_SAD_Group10/0713_Requirements_Code_Agent/code_agent_log.md)。以下為疊代開發與交互紀錄的摘要：

1. **第一次 (建立實作計畫)**：提供專案背景與限制，要求代理規劃實作計畫，建立第一個可操作切片的目錄架構。採用 SPA 與 LocalStorage 技術架構規劃。
2. **第二次 (實作 HTML 骨架與語意標記)**：要求代理建立 `index.html`，具備身分切換與獨立視圖面板（社員、財務、社長、收據查詢），並引入 Inter 與 FontAwesome。
3. **第三次 (實作初始 UI 視覺設計)**：實作毛玻璃玻璃擬態 CSS 樣式，採用 Flexbox/Grid 系統，人工微調了行動端 viewport 及 card padding。
4. **第四次 (實作 LocalStorage 引擎與 20 筆預載資料)**：編寫 JS 初始化邏輯，預載 20 筆社員測試資料，人工修正學號重複問題。
5. **第五次 (實作社員預約與預約編號生成)**：實作社員預約表單送出與 `RES-` 編號自動生成演算法，人工微調日期格式化以確保跨瀏覽器相容性。
6. **第六次 (實作財務搜尋與一鍵收款確認)**：實作小於 0.5s 的模糊搜尋及一鍵確認收款，人工微調事件委派（Event Delegation）防禦 dynamic DOM 監聽器失效。
7. **第七次 (實作社長統計看板與 BR-01 邊界條件)**：計算已繳率與收支總額，**人工修正 BR-01 業務規則**，將逾期金額自社長實收/待收總額中扣除。
8. **第八次 (實作電子收據查驗模組)**：實作收據查詢匹配與動態渲染，人工修正「待繳費」與「已逾期」預約查詢時應提示未繳費，不能產生空白收據。
9. **第九次 (Notion 風格轉換視覺重構)**：依據 `DESIGN-notion.md`，將深色玻璃擬態重構為簡約 Notion 紙質風格，人工修正行動端表格溢出（`overflow-x`）跑版問題。
10. **第十次 (防禦性程式碼與輸入優化)**：過濾學號空白並強制轉大寫，人工修復當搜尋框輸入 Regex 特殊字元時導致模糊搜尋崩潰的 bug。
