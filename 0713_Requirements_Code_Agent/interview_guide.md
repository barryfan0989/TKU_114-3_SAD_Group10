# 訪談大綱 (Interview Guide)

本文件針對「社費預約繳費、一鍵確認與電子收據管理系統」的三大核心利害關係人，設計結構化的訪談大綱，用以萃取系統之功能與非功能性需求。

---

## 1. 訪談角色：一般社員 (General Member)
* **角色與系統關係**：系統的主要繳款預約者與收據查詢者。
* **訪談目的**：確認社員在選擇社費方案、預約、現場繳費、查詢/取得收據時的真實體驗與痛點，並調查是否有逾期失效等行為偏好。

### 訪談問題清單
1. **背景問題**：你平常加入社團時，通常會選擇哪種社費方案（例如半年期或一年期）？你主要是依據什麼來做決定的？
2. **現況流程問題**：你目前在社團中是怎麼完成社費繳款的？可以描述一下從你知道要繳費到拿到憑證的完整過程嗎？
3. **痛點問題**：在繳費的過程中，你遇過最困擾或最花時間的事情是什麼？（例如：找不到幹部、紙本收據弄丟、不知道自己繳了沒？）
4. **例外情況問題**：如果你預約了繳費，但臨時因為期中考忙碌或忘記，超過多久沒繳費，你會覺得這個預約直接作廢是合理的？
5. **資料與通知需求**：當你在線上完成繳款預約後，你希望在畫面上立刻看到哪些資訊，讓你覺得這個預約是成功且安心的？
6. **成功標準問題**：如果有一個新系統可以讓你線上預約並在繳費後下載電子收據，你會期望它的操作流程簡化到什麼程度，才願意主動去用它？

## 2. 訪談角色：財務幹部 (Financial Officer)
* **角色與系統關係**：負責現場收受現金、核對社員身分、一鍵確認收款並發放收據的系統操作者。
* **訪談目的**：釐清幹部在收款、記帳、核對名冊時的流程瓶頸，以及對模糊搜尋、一鍵確認、電子收據經辦人紀錄的具體需求。

### 訪談問題清單
1. **背景問題**：你平均每學期要處理多少位社員的社費收取？每次現場收款大概會花多少時間？
2. **現況流程問題**：社員現場拿現金給你時，你目前是用什麼工具（紙本名冊、Google 試算表、手機備忘錄）記錄的？如何給對方收據？
3. **痛點問題**：在現場大排長龍的收款情境下，最容易出錯的環節是什麼？（例如：找錯零錢、名字太像記錯人、事後核對金額對不上？）
4. **例外情況問題**：如果遇到社員臨時取消預約，或是方案選錯（例如社員現場說要改成一年期但預約是半年期），你希望系統能怎麼協助你快速調整？
5. **權限與責任問題**：為了避免非財務人員誤點或惡意修改已繳費狀態，你認為系統在操作收款確認時，應該留下哪些「經辦人」與時間證據？
6. **成功標準問題**：你希望在搜尋一位前來繳款的社員時，輸入什麼關鍵字（如學號、姓名的一部分、預約編號）就能立刻找到他？一鍵確認收款應該要在幾秒內反應才不會造成卡頓？

## 3. 訪談角色：社長 (Club President)
* **角色與系統關係**：檢視社團財務狀況、催繳未繳費用、評估社團預算與決策的管理者。
* **訪談目的**：確認社長對於整體社費繳交率、實收金額、未繳名單等財務數據的管理維度，以及在課外活動組評鑑時對收據的法規需求。

### 訪談問題清單
1. **背景問題**：身為社長，你通常多久需要向社團幹部或學校課外組匯報一次社團財務狀況？
2. **現況流程問題**：你目前是如何掌握社團收費進度與未繳費社員名單的？需要頻繁詢問財務幹部嗎？
3. **痛點問題**：在了解社團財務時，你最討厭遇到什麼狀況？（例如：帳目不清、報表格式雜亂、無法即時知道有多少預算可用？）
4. **例外情況與稽核問題**：如果發現帳目與實收現金有出入，你通常會如何追溯？你希望電子收據上有哪些資訊以方便你與幹部進行稽核對帳？
5. **資料可視化需求**：在你的管理儀表板上，你最希望一眼看到哪些數據指標？（例如：已收總金額、繳費率百分比、未繳費人數？）
6. **法規與成功標準問題**：學校課外組在社團評鑑時，對於電子收據與財務名冊的真實性與備查格式有什麼要求？你希望系統產出的收據如何滿足這些規範？
