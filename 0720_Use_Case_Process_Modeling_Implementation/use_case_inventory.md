# 使用案例清單 (Use Case Inventory)

本系統依據參與者目標與業務功能，整理出 7 個核心使用案例，並定義其優先順序（MoSCoW）與描述需求。

## 包含 (Include) 與延伸 (Extend) 關係說明

1. **`UC-04 確認收取現付` 包含 (Include) `UC-03 搜尋與篩選預約`**：財務幹部在現場收款時，**必定**要先搜尋到該社員的待繳預約，才能進行一鍵確認。
2. **`UC-02 取消待繳預約` 延伸 (Extend) `UC-01 預約繳費方案`**：在預約成功且狀態為 `待繳費`（特定條件）下，社員或幹部可點擊取消，延伸原預約之合約生命週期。
3. **`UC-05 作廢已繳款收據` 延伸 (Extend) `UC-04 確認收取現付`**：在確認收款成功且開立收據（特定條件）下，因財務誤輸入或學員现场退款，幹部可作廢該收據。
