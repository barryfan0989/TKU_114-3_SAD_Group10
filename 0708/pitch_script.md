# 三分鐘短講提案稿（3-Minute Project Pitch Script）

- **時間分配與講稿結構**

### 1. 專案題目
老師、同學們好，我們是 Group10，我們的專案題目是「面向一般社員、財務幹部與社長之社費預約繳費、一鍵確認與電子收據管理系統」。

### 2. 目標使用者與痛點
在大學社團中，社費收取目前多半依靠人工現金收付與紙本收據。一般社員常搞混「年繳」和「半年繳」的方案，且紙本收據極易遺失，無法隨時核對與佐證個人繳費狀態。而對於財務幹部而言，收款需要逐一與社員約定時間；手動開立收據十分繁瑣，人工對帳也容易因出錯而產生帳務爭議。社長則因缺乏即時數據，難以即時掌握整體收費進度與進行催繳。

### 3. 系統想改善的流程
本系統希望能透過數位化流程改善這些痛點：社員能在線上完成預約繳費並選擇方案，取得唯一的預約編號；財務幹部在收到現金時，只需在後台輸入學號或姓名，即可模糊搜尋並一鍵確認收款；確認收款後，系統會自動生成含時間戳記的電子收據。這不僅實現了社團財務無紙化，也大幅降低了幹部對帳與對時的行政負擔。

### 4. 範圍內與範圍外項目
本期專案的**範圍內**包含：社員註冊與登入、線上選擇社費方案、產生預約編號、登入首頁狀態提醒、財務後台學號/姓名模糊搜尋與一鍵收款確認、自動生成含時間戳記與經辦人的電子收據，以及社長的整體統計儀表板。
而為了在 5 週內順利完成，我們將以下內容列為**範圍外**：不包含真實的線上金流與行動支付、不包含自動網銀對帳、排除校務系統或 SSO 串接、本期不串接外部 Email/簡訊自動通知服務，且原型僅以單機與模擬資料庫方式展示。

### 5. 三個期末原型任務
我們在期末原型中會完整展示三個核心任務：
- **任務一（一般社員選擇方案並產生預約）**：社員登入，選擇繳費方案，系統成功產生唯一的預約編號，狀態設為「待付款」。
- **任務二（財務幹部模糊搜尋與一鍵收款確認）**：幹部拿到現金後，在後台模糊搜尋該社員的學號或姓名，並一鍵確認收款，系統即時將狀態更新為「已繳費」並記錄收款時間。
- **任務三（查驗收據與統計看板）**：社員在個人介面可看到並下載含時間戳記與經辦人的電子收據；社長則可透過統計儀表板，即時掌握已繳/未繳名單與實收繳費率。

### 6. 最大風險與目前應對方式
目前我們面臨的最大風險是「一人開發時程吃緊風險」與「財務幹部操作排斥風險」。我們將透過嚴格限縮範圍為單機/模擬資料庫，並設計極簡的「一鍵確認」與「模糊搜尋」操作介面，以提高幹部使用意願，確保專案能如期且高質量地完成原型。謝謝大家！
